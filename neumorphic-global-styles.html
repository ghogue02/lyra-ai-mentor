Step 1: Add CSS Variables to Your Global Stylesheet

/* Add this to your main CSS file or create a new file: neumorphic-styles.css */ :root { /* Base colors */ --nm-bg-color: #f0f4f8; --nm-text-primary: #2d3748; --nm-text-secondary: #718096; --nm-text-muted: #a0aec0; --nm-accent-color: #5a67d8; --nm-accent-hover: #667eea; /* Shadow colors */ --nm-shadow-dark: rgba(214, 220, 229, 0.8); --nm-shadow-light: rgba(255, 255, 255, 0.8); /* ... all other variables ... */ } /* IMPORTANT: Button styles with proper shadows */ .nm-button { background: var(--nm-bg-color); color: var(--nm-accent-color); box-shadow: var(--nm-shadow-sm); /* ... base button styles ... */ } .nm-button-primary { background: var(--nm-accent-color); color: white; box-shadow: 6px 6px 12px var(--nm-shadow-dark), -6px -6px 12px var(--nm-shadow-light), inset 1px 1px 2px rgba(0, 0, 0, 0.1), inset -1px -1px 2px rgba(255, 255, 255, 0.1); } /* All button variants must include shadows! */

Step 4: Migration Mapping

Class Replacement Guide:

  • .card → .nm-card
  • .btn → .nm-button
  • .btn-primary → .nm-button nm-button-primary
  • .btn-lg → .nm-button nm-button-lg
  • .form-control → .nm-input
  • .avatar → .nm-avatar
  • .badge → .nm-badge
  • .progress → .nm-progress
  • .container → .nm-container (for boxed sections)
Note on buttons: Always combine button modifiers with the base .nm-button class:
<button class="nm-button nm-button-primary"> NOT <button class="nm-button-primary">

Step 5: Responsive Considerations

/* Add these media queries to your CSS */ @media (max-width: 768px) { :root { /* Reduce shadows on mobile for performance */ --nm-shadow-md: 4px 4px 8px var(--nm-shadow-dark), -4px -4px 8px var(--nm-shadow-light); } .nm-card { padding: var(--nm-spacing-sm); } .nm-button { padding: 0.5rem 1rem; } } /* Dark mode support (optional) */ @media (prefers-color-scheme: dark) { :root { --nm-bg-color: #1a202c; --nm-text-primary: #e2e8f0; --nm-shadow-dark: rgba(0, 0, 0, 0.6); --nm-shadow-light: rgba(45, 55, 72, 0.3); } }

Step 7: Testing Checklist

✅ Verify These Elements:

  1. All cards/panels have proper shadows and hover effects
  2. Buttons show pressed state on click
  3. Form inputs have inset shadows
  4. Text contrast meets accessibility standards
  5. Hover states work on all interactive elements
  6. Mobile view maintains readability
  7. Page load performance is acceptable

Final Notes

⚡ Performance Tips:

  • Use CSS transforms instead of position changes for animations
  • Limit the number of elements with complex shadows on mobile
  • Consider using will-change: transform on frequently animated elements
  • Test on low-end devices to ensure smooth performance

🎨 Design Consistency:

  • Always use the CSS variables instead of hard-coded values
  • Maintain consistent spacing using the spacing variables
  • Keep hover states subtle (don't over-animate)
  • Ensure all interactive elements have visible feedback
